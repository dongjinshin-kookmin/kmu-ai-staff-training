# 클리어 그리드 (Clear Grid) 디자인 가이드

## 1. 가이드 개요

**가이드 이름**: 클리어 그리드 (Clear Grid)

**디자인 철학**: 데이터가 스스로 말하게 한다. 장식을 덜어내고 명료한 격자(grid) 위에 수치·차트·표를 정렬하여, 독자가 정보의 위계를 즉시 파악하도록 한다. 모든 시각 요소는 "왜 이 자리에 있는가"라는 질문에 답할 수 있어야 하며, 근거 없는 색·선·여백은 배제한다.

**참고한 디자인 시스템**: Google Material Design 3(M3). 컬러 롤(color role) 개념(primary / on-primary / primary-container), 토널 팔레트(tonal palette) 체계, 타입 스케일(type scale) 위계, 8dp 그리드 시스템을 인쇄물(A4) 환경에 맞게 8pt 그리드로 번안하여 적용한다.

**어울리는 보고서**: 데이터·차트가 다수 포함된 성과 보고서, 통계 중심 결과보고서, 만족도·예산·지표를 다루는 부서 결과보고서. 국민대학교 각 부서의 사업 결과보고서·워크숍 결과보고서처럼 정량 데이터가 핵심 근거로 쓰이는 문서에 최적화되어 있다.

## 2. 디자인 원칙

**원칙 1 — 격자를 신뢰한다.** 모든 요소는 8pt 배수 그리드 위에 정렬한다. 여백·요소 간격·컴포넌트 높이는 8의 배수(8·16·24·32pt)를 기본으로 하며, 미세 조정이 필요한 경우에만 4pt 하프 스텝을 허용한다. 격자를 벗어난 자유 배치는 표지·간지 등 의도된 예외에서만 사용한다.

**원칙 2 — 색은 역할로 쓴다.** 색을 취향이 아니라 기능으로 배정한다. Primary는 핵심 강조와 브랜드 식별, Secondary는 보조 강조, Accent는 KPI·경고 등 시선 유도, Neutral은 본문·괘선·배경에 사용한다. 하나의 색이 여러 의미를 겸하지 않도록 한다.

**원칙 3 — 데이터 잉크를 최대화한다.** Edward Tufte의 데이터-잉크 비율 개념을 따른다. 차트의 격자선·테두리·그림자·3D 효과 등 정보를 담지 않는 잉크는 제거하거나 최소화하고, 데이터 자체(막대·선·수치)에 시각적 무게를 집중한다.

**원칙 4 — 위계는 크기와 굵기로 만든다.** 색만으로 위계를 표현하지 않는다. 제목-부제-본문-캡션의 구분은 우선 타입 스케일(pt)과 굵기(weight)로 확보하고, 색은 보조 수단으로 쓴다. 이로써 흑백 출력에서도 위계가 유지된다.

**원칙 5 — 접근성을 전제로 한다.** 모든 본문 텍스트와 배경의 명도 대비는 WCAG 2.1 AA(4.5:1) 이상을 만족한다. 색각 이상 사용자를 고려해 차트 계열 구분은 색상 단독이 아니라 명도·패턴·직접 레이블을 병행한다.

## 3. 컬러 시스템

### 3.1 코어 팔레트

Primary 슬롯은 **대학 CI 주색상으로 교체 가능한 슬롯**이다. 아래 블루는 기본 제공값이며, 국민대학교 공식 CI 색상값은 본 가이드가 단정하지 않는다. 부서 브랜드 적용 시 홍보팀이 배포한 정확한 CI 색상 코드를 확인하여 Primary(#1565C0) 자리에 대입하고, 나머지 토널 단계는 해당 색을 기준으로 명도만 조정해 재생성한다.

| 롤(Role) | 명칭 | HEX | 용도 |
|---|---|---|---|
| **Primary** | Blue 700 | `#1565C0` | 핵심 강조, 제목 강조선, 주요 막대 (**← CI 교체 슬롯**) |
| On-Primary | White | `#FFFFFF` | Primary 배경 위 텍스트 |
| Primary Container | Blue 100 | `#D3E4FD` | KPI 카드 배경, 강조 박스 |
| **Secondary** | Blue Gray 600 | `#546E7A` | 보조 강조, 부제, 서브 막대 |
| **Accent** | Amber 600 | `#FFB300` | KPI 하이라이트, 목표선, 시선 유도 |
| **Neutral 900** | Ink | `#1A1C1E` | 본문 텍스트 |
| Neutral 600 | Gray | `#5F6368` | 캡션, 보조 텍스트 |
| Neutral 300 | Line | `#DADCE0` | 표 괘선, 구분선 |
| Neutral 50 | Paper | `#F8F9FA` | 표 헤더·zebra 배경 |

### 3.4 차트 카테고리 컬러 (6색)

계열 구분 전용. 순서대로 사용하며, 인접 계열의 명도 차를 확보해 흑백 출력·색각 이상에서도 구분되도록 설계했다.

| 순번 | 명칭 | HEX |
|---|---|---|
| C1 | Blue | `#1565C0` |
| C2 | Teal | `#00838F` |
| C3 | Amber | `#FF8F00` |
| C4 | Purple | `#6A1B9A` |
| C5 | Green | `#2E7D32` |
| C6 | Rose | `#AD1457` |

### 3.5 사용 비율 규칙

- **60 : 30 : 10 규칙** — Neutral(배경·본문) 60%, Primary/Secondary 30%, Accent 10%. Accent가 지면의 10%를 초과하면 강조 기능을 상실하므로 초과를 금지한다.
- 한 페이지에서 차트 카테고리 컬러는 최대 6색까지만 사용한다. 계열이 7개 이상이면 상위 5개 + "기타"로 묶는다.

### 3.6 명도 대비 (WCAG 2.1)

| 조합 | 대비비 | 기준 |
|---|---|---|
| Neutral 900 `#1A1C1E` / Paper `#F8F9FA` | 15.8:1 | AAA 충족 |
| Primary `#1565C0` / White | 5.9:1 | AA 충족 (본문 가능) |
| On-Primary White / Primary `#1565C0` | 5.9:1 | AA 충족 |
| Neutral 600 `#5F6368` / White | 5.9:1 | AA 충족 (캡션 가능) |
| Accent `#FFB300` / White | 1.6:1 | 텍스트 금지 (그래픽·면 채움 전용) |

**규칙**: 본문·캡션은 4.5:1 이상을 반드시 만족한다. Accent(Amber) 계열은 흰 배경 위 텍스트로 쓰지 않으며 면 채움·아이콘에만 사용한다.

## 4. 타이포그래피

### 4.1 서체

- **한글 본문**: Pretendard (SIL Open Font License, 무료). Regular(400)·Medium(500)·SemiBold(600)·Bold(700) 사용.
- **영문·라틴**: Pretendard가 라틴 글리프를 포함하므로 본문은 한 서체로 통일한다.
- **숫자·데이터**: Roboto Mono (Apache License 2.0, 무료). 표의 수치, 차트 축 레이블, KPI 값에 병용하여 자릿수 정렬(tabular)을 확보한다. Pretendard 사용 시에도 숫자에는 `tnum` OpenType 기능을 켜 자간을 고정한다.

### 4.2 타입 스케일

M3 타입 스케일을 인쇄용 pt로 번안했다. 행간(leading)은 pt 값 대비 %로 표기한다.

| 롤 | 용도 | 크기(pt) | 굵기 | 행간(%) | 자간 |
|---|---|---|---|---|---|
| Display | 표지 제목 | 40 | Bold 700 | 115% | -0.5pt |
| Headline | 장 제목(간지) | 28 | Bold 700 | 120% | -0.3pt |
| Title L | 절 제목(H2) | 20 | SemiBold 600 | 130% | -0.2pt |
| Title M | 소제목(H3) | 15 | SemiBold 600 | 140% | 0 |
| Body L | 본문 | 10.5 | Regular 400 | 150% | 0 |
| Body S | 표 내부 본문 | 9.5 | Regular 400 | 140% | 0 |
| Label | 캡션·범례·주석 | 8 | Medium 500 | 130% | +0.2pt |
| Data | KPI 수치 | 32 | Roboto Mono Bold | 100% | -1pt |

### 4.3 위계 규칙

- **규칙 4-1**: 한 지면에서 위계 단계는 최대 4단계(H2 → H3 → 본문 → 캡션)로 제한한다.
- **규칙 4-2**: 제목은 굵기와 크기로 구분하고, 색은 Primary 또는 Neutral 900만 쓴다. 제목에 여러 색을 섞지 않는다.
- **규칙 4-3**: 본문 한 줄 길이는 45~75자(한글 기준 33~40자)를 유지한다. 초과 시 2단 구성을 검토한다.
- **규칙 4-4**: 단락 간격은 본문 크기의 0.75배(약 8pt)를 기본으로 하며 8pt 그리드에 맞춘다.

## 5. 레이아웃·그리드

### 5.1 판형·여백 (A4 세로, 210 × 297mm)

| 항목 | 값 |
|---|---|
| 상단 여백 | 20mm |
| 하단 여백 | 22mm (쪽번호 공간 포함) |
| 안쪽(제본) 여백 | 20mm |
| 바깥쪽 여백 | 16mm |
| 콘텐츠 영역 | 174 × 255mm |

### 5.2 컬럼 그리드

- 기본 **12컬럼** 그리드, 컬럼 간격(gutter) 4mm. 본문 문서는 12컬럼 통합 1단 또는 6+6 2단 구성.
- 데이터 페이지: 좌측 8컬럼(차트) + 우측 4컬럼(해설) 조합을 권장한다.

### 5.3 베이스라인 그리드

- 본문 베이스라인 간격 **4.5mm**(≈ Body L 10.5pt × 150% 행간)에 모든 텍스트 라인을 스냅하고, 8pt(≈ 2.82mm) 인쇄 그리드를 겹쳐 요소 상·하단을 정렬한다.

### 5.4 쪽번호·러닝헤드

- 쪽번호: 하단 바깥쪽, Label 8pt Neutral 600, 표지·간지는 생략. 러닝헤드(상단): 장 제목 Label 8pt Neutral 600, 상단 12mm 지점.

## 6. 컴포넌트 스타일

### 6.1 표(Table)

| 항목 | 규격 |
|---|---|
| 헤더 배경 | Primary Container `#D3E4FD` |
| 헤더 텍스트 | Neutral 900, SemiBold 600, 9.5pt |
| 셀 상하 패딩 | 6pt |
| 셀 좌우 패딩 | 8pt |
| 가로 괘선 | Neutral 300 `#DADCE0`, 0.5pt |
| 세로 괘선 | **원칙적으로 없음** (데이터 잉크 최소화) |
| Zebra 배경 | 짝수 행 Paper `#F8F9FA` (5행 이상일 때만 적용) |
| 숫자 정렬 | 우측 정렬, Roboto Mono, 소수점·자릿수 통일 |
| 텍스트 정렬 | 좌측 정렬 |

- **규칙 6-1**: 표는 상단·하단에 0.75pt Neutral 900 경계선을 두고, 내부는 가로 괘선만 사용하는 "하프라인(horizontal-only)" 스타일을 기본으로 한다.

### 6.2 차트 공통 규칙

- **규칙 6-2**: 3D 효과·그림자·그라디언트를 금지한다. 면은 단색으로 채운다.
- **규칙 6-3**: 배경 격자선은 Neutral 300 0.25pt 가로선만 사용하고 세로 격자선은 생략한다.
- 축선: Neutral 600 0.5pt. 축 레이블: Roboto Mono 8pt Neutral 600.
- 데이터 레이블: 가능하면 범례 대신 계열 끝에 직접 레이블(direct labeling)을 배치한다.

#### 막대 차트

- 막대 폭 : 간격 = 2 : 1, 세로 막대 폭 최소 6mm. 단일 계열은 Primary `#1565C0`, 다계열은 카테고리 컬러 C1~C6 순.
- 모서리 직각(radius 0), 값 레이블은 막대 상단 바깥 2pt 위 Body S 9.5pt. 기준선(0)은 반드시 표시하며 축을 절단(truncate)하지 않는다.

#### 선 차트

- 선 두께 2pt, 데이터 포인트 원 지름 4pt, 계열 3개 이하 권장. 강조 계열만 Primary, 나머지는 Neutral 600 또는 카테고리 컬러.
- 목표선·평균선은 Accent `#FFB300` 1pt 점선(dash 4pt / gap 2pt).

#### 도넛 차트

- 내경 : 외경 = 0.6 : 1(두께 지름의 20%), 조각 최대 5개(+기타), 조각 사이 흰 간격 1pt.
- 중앙 공백에 총합·핵심 수치를 Data 스타일로 배치. 원형(파이) 대신 도넛을 기본으로 하며 3D 파이는 금지한다.

### 6.3 KPI 카드

| 항목 | 규격 |
|---|---|
| 카드 크기 | 폭 54mm × 높이 32mm (3열 배치 기준) |
| 배경 | Primary Container `#D3E4FD` 또는 Paper `#F8F9FA` |
| 모서리 | radius 8pt |
| 내부 패딩 | 12pt |
| 수치 | Data 스타일 Roboto Mono Bold 32pt, Primary |
| 라벨 | Label 8pt Neutral 600, 수치 하단 4pt |
| 증감 배지 | Success/Error 색, 8pt, 삼각 아이콘 병기(△/▽) |

### 6.4 인용·콜아웃 박스

- 배경 Blue 50 `#E8F0FE`, 좌측 4pt Primary 세로 바.
- 내부 패딩 12pt, 본문 Body L 10.5pt, 상단에 Label로 유형 표기("참고", "주의" 등).
- 주의형은 좌측 바를 Warning `#F9A825`로 교체한다.

### 6.5 아이콘

- Material Symbols(Outlined, Apache 2.0, 무료), 굵기 400, 그리드 24pt. 본문 인라인 16pt, 색은 동반 텍스트와 동일. 장식용 다색 아이콘 금지.

### 6.6 사진 처리

- 모서리 radius 8pt, 테두리·그림자 없음. 최소 300dpi(인쇄 기준), 색조 보정은 채도 -5% 통일. 캡션은 사진 하단 4pt, Label 8pt Neutral 600, "그림 N. 설명" 형식.

## 7. 표지·목차·간지 디자인 지침

### 7.1 표지

- 상단 24mm 지점: 발간 기관명(국민대학교 ○○부서) Label 8pt Neutral 600.
- 수직 중앙 상단: Primary 색 굵은 가로 바(폭 60mm × 높이 6pt) 하나로 브랜드 앵커 배치.
- 그 아래 8pt 간격: 보고서 제목 Display 40pt Bold, Neutral 900, 최대 2줄.
- 제목 하단 16pt: 부제 Title L 20pt SemiBold Secondary.
- 하단 22mm 지점: 발간 연월·문서번호 Label 8pt. 배경은 Paper 단색, 이미지 사용 시 하단 1/3에만 배치하고 상단 2/3는 여백으로 비운다.

### 7.2 목차

- 좌측 4컬럼: "목차 / Contents" Headline 28pt.
- 우측 8컬럼: 장 번호(Roboto Mono, Primary) + 장 제목(Title M 15pt) + 점선 리더 + 쪽번호(Roboto Mono 우측 정렬).
- 항목 간 세로 간격 12pt. 대분류만 굵게, 소분류는 Regular로 들여쓰기 8pt.

### 7.3 간지(장 표지)

- 상단 40mm 지점: 장 번호를 Data 스타일 Roboto Mono 32pt Primary로 큼직하게.
- 그 아래 8pt: 장 제목 Headline 28pt Bold Neutral 900.
- 하단: Primary 가로 바(폭 40mm × 4pt). 전체 배경 Paper, 정보 밀도를 낮춰 휴지(pause) 지면으로 기능하게 한다.

## 8. 적용 예시 — '워크숍 결과보고서'

국민대학교 부서가 개최한 워크숍의 결과보고서에 본 가이드를 적용하는 페이지별 구성 지침이다.

**표지 (1p)**: 제목 "2026 ○○ 워크숍 결과보고서" Display 40pt. Primary 브랜드 바 배치(CI 교체 슬롯 색 적용). 하단에 "국민대학교 ○○부서 · 2026.07" Label 8pt.

**개요 (2p)**: 상단 Title L 20pt "1. 워크숍 개요". 좌측 8컬럼 본문(목적·주제·주최), 우측 4컬럼에 KPI 카드 3개 세로 배치 — "참가자 82명", "만족도 4.4/5", "프로그램 6개". 카드 수치는 Roboto Mono 32pt Primary.

**일정표 (3p)**: 하프라인 표 스타일. 헤더 배경 Primary Container. 열 구성 "시간 | 세션 | 발표자 | 장소". 시간 열은 Roboto Mono 우측 정렬. 5행 이상이면 짝수 행 Paper zebra 적용. 셀 상하 패딩 6pt.

**만족도 차트 (4p)**: 좌 8컬럼에 항목별 만족도 가로 막대 차트(5점 척도), 단일 계열 Primary. 목표선 4.0을 Accent 점선으로 표시. 각 막대 끝에 값 레이블 직접 표기. 우 4컬럼에 도넛 차트 1개(전반 만족도 분포: 매우만족/만족/보통/불만족 4조각, 카테고리 컬러 C1~C4), 중앙 공백에 평균 "4.4" Data 스타일.

**예산표 (5p)**: 하프라인 표, "항목 | 예산(원) | 집행(원) | 집행률(%)". 금액은 Roboto Mono 우측 정렬, 천단위 콤마. 집행률 열은 미달 시 Error, 초과 없음 시 Neutral 900. 합계 행은 상단 0.75pt Neutral 900 경계선 + SemiBold. 표 하단에 "※ 단위: 원, VAT 포함" Label 8pt 주석.

## 9. 발간 전 점검 체크리스트

1. 모든 요소가 8pt 그리드에 정렬되어 있는가.
2. 본문·캡션의 명도 대비가 WCAG AA(4.5:1) 이상인가.
3. Primary 슬롯에 확인된 CI 색상(또는 기본 블루)이 일관 적용되었는가.
4. Accent(Amber)를 흰 배경 위 텍스트로 잘못 쓰지 않았는가.
5. 차트에 3D·그림자·세로 격자선이 없는가.
6. 막대·선 차트의 기준선(0)이 절단되지 않았는가.
7. 표가 하프라인(가로 괘선만) 스타일을 지키는가.
8. 숫자가 Roboto Mono로 자릿수·소수점 정렬되어 있는가.
9. 사진이 300dpi 이상이고 캡션·출처가 표기되었는가.
10. 한 페이지 내 차트 계열이 6색을 초과하지 않는가.
